# 써클커넥션 Wiki · 실제 공유 버전 설치 가이드

이 가이드대로 하면 **모든 직원이 같은 문서를 함께 보고**, **회사 구글 계정으로 로그인**하는 진짜 위키가 완성됩니다. 데이터는 Supabase라는 무료 데이터베이스에 저장되어 직원 간에 공유됩니다.

개발 지식이 없어도 따라 할 수 있도록 클릭 순서로 적었습니다. 막히는 부분이 있으면 그 단계 번호를 알려주세요.

전체 순서: ① Supabase 만들기 → ② 데이터베이스 설치 → ③ 구글 로그인 연결 → ④ 열쇠(키) 입력 → ⑤ GitHub·Vercel 배포 → ⑥ 도메인 연결.

> 참고: index.html은 키를 넣기 전까지 **데모 모드**로 동작합니다(이 브라우저에만 저장). 아래를 마치면 자동으로 **실제 공유 모드**로 전환됩니다.

---

## 준비물

회사 Google Workspace 관리자 권한(대표님 보유), GitHub 계정, Vercel 계정, imweb 도메인. 모두 이미 갖고 계십니다.

---

## 1단계. Supabase 프로젝트 만들기 (무료)

Supabase는 데이터를 보관하고 로그인을 처리해주는 서비스입니다.

1. supabase.com 접속 → **Start your project** → GitHub 계정으로 로그인
2. **New project** 클릭
3. 입력값: Name `circle-wiki`, Database Password는 아무 강력한 비밀번호 입력 후 **따로 메모**(나중에 거의 쓸 일 없지만 보관), Region은 **Northeast Asia (Seoul)** 선택
4. **Create new project** 클릭 → 1~2분 기다리면 생성 완료

---

## 2단계. 데이터베이스 설치 (SQL 한 번 실행)

1. 왼쪽 메뉴에서 **SQL Editor** 클릭 → **New query**
2. 같이 드린 **`supabase-schema.sql`** 파일을 메모장 등으로 열어 **전체 복사**
3. SQL Editor에 붙여넣고 우측 **Run** 클릭
4. "Success" 메시지가 뜨면 테이블·보안·샘플 문서가 모두 설치된 것입니다

> 회사 도메인이 ccon.co.kr가 아니거나 대표 이메일이 nate@ccon.co.kr가 아니면, SQL 파일의 6번 항목에 있는 `v_owner`, `v_allow` 값을 먼저 바꾼 뒤 실행하세요.

---

## 3단계. 구글 로그인 연결

회사 구글 계정으로만 로그인되게 만드는 단계입니다. Supabase와 Google을 서로 연결합니다.

### 3-1. 구글에서 로그인 열쇠 발급

1. console.cloud.google.com 접속(회사 계정) → 프로젝트 **새로 만들기** → 이름 `Circle Wiki`
2. **API 및 서비스** → **OAuth 동의 화면** → User Type을 **내부(Internal)**로 선택 → 저장 (내부로 해야 심사·비용이 없습니다)
3. 앱 이름 `써클커넥션 Wiki`, 지원 이메일 입력 후 저장
4. **사용자 인증 정보** → **사용자 인증 정보 만들기** → **OAuth 클라이언트 ID** → 유형 **웹 애플리케이션**
5. **승인된 리디렉션 URI**에 아래 주소를 추가합니다. (★ 매우 중요)

   ```
   https://<프로젝트참조>.supabase.co/auth/v1/callback
   ```

   `<프로젝트참조>` 부분은 Supabase의 **Project Settings → API → Project URL**에 나오는 주소(`https://abcdxyz.supabase.co`)의 `abcdxyz` 부분입니다. 즉 최종 형태는 `https://abcdxyz.supabase.co/auth/v1/callback`.
6. **만들기**를 누르면 **클라이언트 ID**와 **클라이언트 보안 비밀(Secret)** 두 개가 나옵니다. 복사해 둡니다.

### 3-2. Supabase에 구글 연결

1. Supabase 왼쪽 **Authentication** → **Sign In / Providers**(또는 Providers) → **Google** 선택
2. **Enable** 켜고, 방금 받은 **Client ID**와 **Client Secret**을 붙여넣기 → **Save**

### 3-3. 접속 주소 등록

1. Supabase **Authentication** → **URL Configuration**
2. **Site URL**에 `https://wiki.ccon.co.kr` 입력
3. **Redirect URLs**에 아래를 추가(Add URL):
   - `https://wiki.ccon.co.kr/**`
   - (테스트용 임시주소) `https://circle-wiki.vercel.app/**`
4. 저장

## 4단계. 열쇠(키)를 index.html에 입력

1. Supabase **Project Settings → API** 화면을 엽니다.
2. 거기서 두 가지를 복사합니다: **Project URL** 과 **anon public** 키(긴 문자열).
3. `index.html` 파일을 메모장으로 열어 위쪽 **CONFIG** 부분을 찾아 아래처럼 채웁니다.

```js
const CONFIG = {
  SUPABASE_URL:      'https://abcdxyz.supabase.co',   // 복사한 Project URL
  SUPABASE_ANON_KEY: 'eyJhbGciOi...',                 // 복사한 anon public 키
  ALLOWED_DOMAIN:    'ccon.co.kr',
  OWNER_EMAIL:       'nate@ccon.co.kr',
  SERVICE_NAME:      '써클커넥션 Wiki'
};
```

4. 저장합니다. 이 두 값을 넣는 순간부터 실제 공유 모드로 동작합니다.

> 안심하셔도 되는 점: anon 키는 외부에 공개되어도 되는 "공개용 열쇠"입니다. 실제 데이터 보호는 2단계에서 설치한 보안 규칙(RLS)이 담당하므로, 이 키가 코드에 들어가도 안전합니다.

---

## 5단계. GitHub · Vercel 배포

1. GitHub에 새 비공개 저장소(`circle-wiki`)를 만들고 `index.html`을 업로드합니다.
   (이미 데모를 올리셨다면 같은 파일을 덮어쓰면 됩니다.)
2. Vercel → Add New → Project → 그 저장소 Import → Deploy
3. 코드를 수정해 GitHub에 다시 올릴 때마다 Vercel이 자동으로 새로 배포합니다.

> 회사 업무용은 Vercel **Pro($20/월)** 대상입니다. 먼저 무료로 확인 후 정식 운영 시 전환하세요.

---

## 6단계. 도메인 wiki.ccon.co.kr 연결

(자세한 화면 순서는 함께 드린 `배포-가이드.md`의 3단계와 동일합니다.)

요약하면, Vercel 프로젝트 **Settings → Domains**에 `wiki.ccon.co.kr`을 추가하고, **아임웹 DNS**에 **CNAME 레코드**(호스트 `wiki`, 값 `cname.vercel-dns.com`)를 한 줄 추가하면 됩니다. 보안 인증서(HTTPS)는 Vercel이 자동 발급합니다.

연결 후, 혹시 3-3단계에서 임시 주소만 넣었다면 Supabase URL Configuration의 Site URL과 Redirect URLs가 `https://wiki.ccon.co.kr`로 되어 있는지 다시 확인하세요.

---

## 첫 로그인과 운영 방법

1. 배포된 주소(`https://wiki.ccon.co.kr`)에 접속해 **대표님 구글 계정**으로 먼저 로그인하세요. 자동으로 **총괄 관리자**가 됩니다.
2. 다른 직원이 구글로 로그인하면 **"승인 대기"** 상태가 됩니다.
3. 대표님이 우측 상단 **관리자 → 가입 승인** 탭에서 승인하면 그 직원이 이용할 수 있습니다.
4. 직원이 글을 올리면 **승인 대기**로 들어오고, 관리자가 **콘텐츠 승인** 탭에서 승인해야 전체 공개됩니다.

이 모든 데이터는 이제 Supabase에 저장되어 **모든 직원이 동일하게** 보게 됩니다.

## 자주 막히는 곳

로그인 후 "회사 계정만 이용할 수 있습니다"가 뜨면, 3단계의 구글 도메인이 회사 도메인과 일치하는지, SQL의 `v_allow` 값이 맞는지 확인하세요. "승인 대기" 화면에서 멈추면 정상이며, 관리자가 승인하면 됩니다. 구글 버튼을 눌렀는데 오류가 나면, 3-1의 리디렉션 URI(`.../auth/v1/callback`)와 3-3의 Redirect URLs를 다시 확인하세요. 이 세 가지가 대부분의 원인입니다.
