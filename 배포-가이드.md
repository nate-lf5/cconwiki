# 써클커넥션 Wiki 배포 가이드

이 문서 하나만 순서대로 따라 하시면 `https://wiki.ccon.co.kr` 주소로 사내 위키를 띄우고, 회사 구글 계정으로 로그인까지 켤 수 있습니다. 개발 지식이 없어도 따라 하실 수 있도록 클릭 순서 위주로 적었습니다.

배포 대상 파일은 **`index.html`** 하나입니다. (기존 `ccon-wiki.html`은 구버전이니 삭제하셔도 됩니다.)

전체 순서는 다음과 같습니다: ① GitHub에 파일 올리기 → ② Vercel로 배포 → ③ 도메인 연결 → ④ 구글 로그인 켜기.

---

## 1단계. GitHub에 파일 올리기

GitHub은 코드를 보관하는 창고입니다. 이미 계정이 있고 Vercel과 연동되어 있으니 파일만 올리면 됩니다.

1. github.com 접속 후 우측 상단 **+** → **New repository** 클릭
2. 이름을 `circle-wiki` 등으로 입력하고, **Private**(비공개) 선택 후 **Create repository**
3. 만들어진 화면에서 **uploading an existing file** 링크 클릭
4. `index.html` 파일을 끌어다 놓고(드래그) **Commit changes** 클릭

이제 창고에 파일이 올라갔습니다.

---

## 2단계. Vercel로 배포하기

Vercel은 그 창고의 파일을 실제 웹사이트로 띄워주는 서비스입니다.

1. vercel.com 로그인 → **Add New...** → **Project**
2. 방금 만든 `circle-wiki` 저장소 옆 **Import** 클릭
3. 설정은 건드릴 것 없이 **Deploy** 클릭
4. 1분 내로 `circle-wiki.vercel.app` 같은 임시 주소가 생성됩니다. 눌러서 화면이 뜨는지 확인하세요.

> 참고: 회사 업무용은 Vercel 라이선스상 **Pro 플랜($20/월)** 대상입니다. 처음엔 무료로 띄워 확인하시고, 정식 운영 시 Pro로 올리시면 됩니다.

---

## 3단계. 도메인 wiki.ccon.co.kr 연결하기

지금부터가 핵심입니다. `wiki.ccon.co.kr`라는 주소가 Vercel 사이트를 가리키도록 연결합니다. 두 군데에서 각각 한 번씩 설정합니다.

### 3-1. Vercel에 도메인 등록

1. Vercel 프로젝트 → 상단 **Settings** → 왼쪽 **Domains**
2. 입력칸에 `wiki.ccon.co.kr` 입력 후 **Add**
3. Vercel이 "이 CNAME 값을 도메인 관리처에 추가하세요"라는 안내를 보여줍니다. 보통 값은 **`cname.vercel-dns.com`** 입니다. 이 값을 복사해 둡니다. (화면에 다른 값이 보이면 그 값을 그대로 쓰세요.)

### 3-2. 아임웹(imweb)에서 DNS 레코드 추가

아임웹에서 도메인을 사셨으니, 아임웹의 DNS(도메인 레코드) 설정에서 한 줄을 추가합니다.

1. 아임웹 관리자 로그인 → **도메인** 메뉴 → 해당 도메인의 **DNS 레코드 설정**(또는 레코드 관리) 화면으로 이동
2. **레코드 추가**를 누르고 아래처럼 입력합니다.

| 항목 | 입력값 |
|---|---|
| 레코드 종류(Type) | **CNAME** |
| 호스트/이름(Host) | **wiki** |
| 값/대상(Value) | **cname.vercel-dns.com** |
| TTL | 기본값 (예: 3600) |

3. 저장합니다.

> 입력 시 주의: 호스트 칸에 `wiki.ccon.co.kr` 전체가 아니라 앞부분 **`wiki`만** 넣습니다. 아임웹 화면 구성이 가비아/카페24와 조금 다를 수 있는데, 같은 화면을 못 찾으시면 아임웹 채팅 상담에 "외부 호스팅(Vercel)으로 wiki 서브도메인을 CNAME 연결하고 싶다"고 말씀하시면 안내해 줍니다.

설정 후 연결이 완료되기까지 보통 몇 분에서 최대 24시간이 걸립니다. Vercel Domains 화면에 초록색 체크가 뜨면 완료이고, 그때부터 `https://wiki.ccon.co.kr`로 접속됩니다. (보안 인증서 HTTPS는 Vercel이 자동으로 처리합니다.)

## 4단계. 구글 로그인(OAuth) 켜기

회사 Google Workspace 계정으로만 로그인되게 만드는 단계입니다. 대표님은 Workspace 관리자 권한이 있으니 직접 하실 수 있습니다. 내부 전용이라 구글 심사나 비용이 없습니다.

### 4-1. Google Cloud에서 OAuth 동의 화면 설정

1. console.cloud.google.com 접속 (회사 계정으로 로그인)
2. 상단에서 프로젝트 **새로 만들기** → 이름 `Circle Wiki`
3. 왼쪽 메뉴 **API 및 서비스** → **OAuth 동의 화면**
4. User Type(사용자 유형)을 반드시 **내부(Internal)**로 선택 → 저장
   - 이것이 핵심입니다. 내부로 설정해야 ccon.co.kr 직원만 쓸 수 있고 심사가 면제됩니다.
5. 앱 이름 `써클커넥션 Wiki`, 지원 이메일 입력 후 저장

### 4-2. OAuth 클라이언트 ID 만들기

1. 왼쪽 **사용자 인증 정보(Credentials)** → **사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
2. 애플리케이션 유형: **웹 애플리케이션**
3. **승인된 자바스크립트 원본(Authorized JavaScript origins)**에 아래 두 개를 추가합니다.
   - `https://wiki.ccon.co.kr`
   - (테스트용으로) `https://circle-wiki.vercel.app` 같은 임시 주소
4. **만들기**를 누르면 **클라이언트 ID**가 나옵니다. (예: `1234...-abcd.apps.googleusercontent.com`) 복사해 둡니다.

### 4-3. 파일에 클라이언트 ID 붙여넣기

1. `index.html` 파일을 열어 위쪽 **CONFIG** 부분을 찾습니다.
2. 아래처럼 따옴표 안에 복사한 클라이언트 ID를 붙여넣습니다.

```js
const CONFIG = {
  GOOGLE_CLIENT_ID: '여기에-복사한-클라이언트-ID-붙여넣기',
  ALLOWED_DOMAIN: 'ccon.co.kr',
  SERVICE_NAME: '써클커넥션 Wiki'
};
```

3. 저장한 뒤, 1단계처럼 GitHub에 다시 올리면(같은 파일 덮어쓰기) Vercel이 자동으로 새로 배포합니다.

이제 로그인 화면에 진짜 "Google로 로그인" 버튼이 나타나고, ccon.co.kr 계정만 입장할 수 있습니다.

## 데모로 미리 확인하기

클라이언트 ID를 넣기 전에도 화면과 기능은 모두 동작합니다. 로그인 화면의 회색 구글 버튼을 누르면 관리자(Nathan)로 바로 입장해 둘러보실 수 있습니다. 이메일 로그인은 `nate@ccon.co.kr` / 비밀번호 `admin` 입니다.

## 중요한 한 가지 (꼭 읽어주세요)

지금 버전은 **로그인과 화면, 기능을 모두 갖춘 완성형 시안**이지만, 입력한 문서 데이터는 **각자 브라우저 안에만 저장**됩니다. 즉 김지원 씨가 올린 글을 이민호 씨 화면에서는 볼 수 없습니다. 여러 직원이 같은 문서를 공유하려면 뒤에서 데이터를 모아주는 **데이터베이스(예: Supabase)**를 붙이는 추가 개발이 한 번 필요합니다.

따라서 이 단계의 활용법은 두 가지입니다. 첫째, 사내에 "이런 위키를 만들 겁니다" 하고 디자인과 기능을 시연하는 용도. 둘째, 외주 개발사에 "이 화면 그대로, 데이터 공유 기능만 붙여 완성해 달라"고 견적을 의뢰하는 기준 자료. 시안이 구체적이라 견적과 일정이 훨씬 정확해집니다.

원하시면 이 시안을 토대로 한 **외주 개발 견적 요청서(RFP)**도 만들어 드릴 수 있습니다.
