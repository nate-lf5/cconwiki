# Claude 프롬프트 작성 가이드

회사 내 모든 프로젝트와 업무에서 Claude를 효과적으로 활용하기 위한 지침입니다.

---

## 1. 기본 원칙

### 명확함 > 친절함
- 짧고 명확하게 지시하세요.
- "문서를 작성해줘"보다 "마케팅 전략 문서 작성, 분량 A4 2장, 타겟은 30대 여성, 형식은 마크다운"

### 컨텍스트 제공
- 배경 정보, 목표, 제약조건을 함께 제공하세요.
- Claude가 상황을 이해할수록 더 나은 결과를 제공합니다.

### 결과물 형식 명시
- 원하는 결과 형태를 구체적으로 지정하세요.
- 예: "CSV 파일", "마크다운 표", "JSON 형식", "엑셀 시트" 등

---

## 2. 효과적인 프롬프트 구조

### 기본 템플릿

```
[작업 명령어]: [구체적인 요청]

[배경 정보]
- 목적: [왜 이 작업을 하는가]
- 대상: [누가 사용하는가]
- 제약조건: [시간, 예산, 규격 등 제한사항]

[요구사항]
- 포함할 내용: [반드시 들어가야 할 항목]
- 제외할 내용: [빼야 할 항목]
- 형식: [결과물 형태]

[추가 지시사항]
- 톤: [격식적/캐주얼/전문적]
- 길이: [분량이나 세부 수준]
- 참고자료: [첨부 파일이나 참고 링크]
```

---

## 3. 작업 유형별 가이드

### A. 문서 작성 (보고서, 제안서 등)

**좋은 예:**
```
이벤트 제안서 작성해줘.

배경: 새로운 클라이언트를 위한 제안
대상: C급 임원진 (의사결정자)
분량: A4 3~4장
포함사항: 이벤트 개요, 예상 효과, 예산, 일정

형식: 마크다운 (Word 변환 예정)
톤: 전문적이고 자신감 있게
```

**나쁜 예:**
```
제안서 좀 만들어줘
```

---

### B. 데이터 분석 및 정리

**좋은 예:**
```
다음 이벤트 참가자 데이터를 분석해줘.

데이터: [파일명 또는 데이터 붙여넣기]
분석 항목:
- 연령대별 참가율
- 직종별 분포
- 지역별 현황
- 이전 행사 재참가율

결과물: CSV로 정리된 요약 표 + 주요 인사이트 3가지
```

---

### C. 계획 및 일정 수립

**좋은 예:**
```
6월 마케팅 캠페인 일정을 세워줘.

조건:
- 예산: 500만원
- 팀 인원: 3명
- 목표: SNS 팔로워 20% 증가
- 주요 채널: 인스타그램, 블로그

출력: 주간 단위 태스크 리스트 (마크다운 표 형식)
```

---

### D. 검토 및 피드백

**좋은 예:**
```
이 이벤트 기획안을 검토해줘.

평가 기준:
1. 목표 달성 가능성 (현실성)
2. 예산 효율성
3. 참가자 만족도 예상도
4. 운영 난이도

출력: 
- 강점 3가지
- 개선 필요 영역 3가지
- 우려사항과 해결책
```

---

## 4. 프롬프트 작성 체크리스트

프롬프트를 보내기 전에 확인하세요:

- [ ] 목표가 명확한가? (무엇을 원하는가)
- [ ] 배경 정보가 충분한가? (왜 이 작업이 필요한가)
- [ ] 대상 청중이 지정되었는가? (누가 이 결과물을 사용하는가)
- [ ] 결과물 형식이 구체적인가? (어떤 형태로 받고 싶은가)
- [ ] 제약조건이 명시되었는가? (시간, 분량, 예산 등)
- [ ] 톤과 스타일이 지정되었는가? (격식 수준, 작성 방식)

## 5. 자주하는 실수와 개선 방법

### 실수 1: 너무 추상적인 요청
**X** "좋은 이벤트 아이디어 줘"
**O** "20대 직장인 여성 대상, 8월, 온라인 기반, 참가비 0~5만원, 회사 문화 공유 목표 이벤트 아이디어 5가지"

### 실수 2: 참고 자료 없이 결과물 요청
**X** "마케팅 보고서 작성해"
**O** "6월 마케팅 캠페인 결과 분석 보고서 작성. [첨부 파일: 캠페인 데이터], 형식: 엑셀"

### 실수 3: 예상과 다른 결과 받기
**X** "예산안 만들어줘"
**O** "이벤트 예산안: 참가자 500명, 장소는 강남 컨퍼런스홀, 주요 항목은 장소비/카테링/운영비/마케팅비, 형식은 엑셀 시트, 참고할 지난해 예산: [파일명]"

## 6. 응답이 마음에 들지 않을 때

### 방법 1: 추가 지시
```
지금 내용에서 XXX 부분을 더 자세하게 다시 작성해줘.
```

### 방법 2: 방향 수정
```
지금 방향이 너무 형식적이야. 더 캐주얼하고 창의적으로 다시 작성해줘.
```

### 방법 3: 구체적 피드백
```
이 부분은 우리 회사 상황에 맞지 않아. 
[구체적인 이유]를 반영해서 수정해줘.
```

## 7. 팁과 베스트 프랙티스

### 팁 1: 한 번에 여러 작업은 피하기
**X** "예산 분석도 하고, 일정도 짜고, 보고서도 만들어줘"
**O** 각각 따로 요청하기

### 팁 2: 수정 사항은 명확하게
대신 "이건 아니야"라고 하기보다, 구체적으로 말하세요.

**X** "이 안은 좋지 않아"
**O** "이 안은 예산이 초과돼. 15% 줄여서 다시 해줘"

### 팁 3: 예시 제공하기
원하는 결과물의 예시가 있다면 함께 제공하세요. Claude가 스타일을 더 정확히 맞춥니다.

### 팁 4: 반복 작업은 명확하게
"매주 이런 분석을 하려면..." 같이 맥락을 주면 더 나은 템플릿을 만들어줍니다.

## 8. 이벤트 에이전시 특화 프롬프트 예시

### 예시 1: 이벤트 기획
```
D2D(Data to Decision) 컨퍼런스 기획안 작성해줘.

기본 정보:
- 규모: 500명 규모
- 예상 참가층: 마케팅 담당자, 데이터 분석가
- 날짜: 9월 15일 (금)
- 예산: 1,500만원

포함사항:
1. 이벤트 개요 및 목표
2. 예상 참가자 특성
3. 프로그램 구성 (기조연설, 세션, 네트워킹 등)
4. 장소 및 운영 방안
5. 예산 배분안
6. 성공 지표

출력: 마크다운 형식, A4 5~6장 분량
```

### 예시 2: 마케팅 전략
```
이벤트 마케팅 계획 수립해줘.

이벤트 정보:
- 제목: 스타트업 네트워킹 밤
- 대상: 초기 스타트업 창업자, 투자자
- 예상 참가자: 300명
- 예산: 200만원

요청사항:
1. 타겟층별 마케팅 전략
2. 채널별 실행 방안 (SNS, 이메일, 직접 섭외 등)
3. 주간 실행 일정
4. 성과 측정 지표

출력: 엑셀 파일 (전략 + 일정 시트)
```

## 9. 빠른 참조

**핵심 요소 5가지:**
1. **무엇을** 원하는가 (결과물)
2. **왜** 필요한가 (목적)
3. **누가** 사용하는가 (대상)
4. **어떻게** 만들어야 하는가 (형식, 톤)
5. **언제까지** 필요한가 (마감일, 서두르는 정도)

이 5가지만 명확하면 Claude의 응답 품질이 크게 향상됩니다.
